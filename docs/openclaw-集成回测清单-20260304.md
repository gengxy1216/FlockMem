# OpenClaw 集成回测清单（2026-03-04）

本清单用于 Task02/Task03 后的 OpenClaw 侧回测，目标是验证：

1. primary 模型继承/覆盖策略正确。
2. sender/channel/group 路由正确。
3. ACL 回退策略生效且不中断主流程。
4. 时延未超预算。

## 1. 前置准备

1. 启动 MiniMem 服务，确认 `GET /health` 正常。
2. 准备两个 OpenClaw agent（如 `agent-a`、`agent-b`）。
3. 确保 `~/.openclaw/openclaw.json` 可读写。

## 2. 安装与同步

Windows:

```powershell
powershell -ExecutionPolicy Bypass -File integrations/openclaw-plugin/install.ps1 -ForcePrimarySync
```

Linux/macOS:

```bash
bash integrations/openclaw-plugin/install.sh --force-primary-sync
```

检查点：

1. `plugins.entries.minimem-memory.config.primaryModelSyncStatus` 为 `applied` 或 `skipped_manual_override`。
2. `senderMap/channelGroupMap/sharePolicy` 已写入配置（允许为空对象）。

## 3. 路由与隔离回测

### 用例A：共享组可召回

1. 给 `agent-a`、`agent-b` 配置同一 `group_id` 或同一 `channelGroupMap` 映射。
2. `agent-a` 写入一条可检索记忆。
3. `agent-b` 使用相同 query 检索，应可召回。

### 用例B：私有组不可串读

1. 将 `agent-a`、`agent-b` 映射到不同 `group_id`。
2. `agent-a` 写入记忆后，`agent-b` 同 query 检索，不应召回该条。

### 用例C：ACL 拒绝后回退

1. 在 `sharePolicy` 中为目标组设置 deny（不包含当前 agent）。
2. 发起读取/写入，确认请求不中断。
3. 检查写入内容 metadata 是否带 `route_acl_fallback=true`。

### 用例D：agent/channel 动态增删

1. 运行时新增一个 agent（或 channel）但先不配映射，请求应自动回退默认路由且不中断。
2. 运行时补充 `senderMap/channelGroupMap` 映射，再发起一次请求，应命中新映射（无需重装插件）。
3. 运行时删除一个 agent（或 channel）后再次请求，不应报错；应继续按默认路由工作。
4. 若 `sharePolicy` 未覆盖新增 agent，行为应为 `aclFallback=true` 且回退默认组。

## 4. Hook 回测

1. `before_agent_start`：启动 agent 前确认有记忆上下文注入。
2. `agent_end`：结束后确认自动写入 `dialogue/context_compression`。
3. 检查 metadata 是否含 `agent_id/channel/task_id/trace_id`。
4. 压缩治理：分别验证 `compressionMode=truncate/hybrid`，并确认 `hybrid` 失败时回退 `truncate` 且不中断。

## 5. 性能回测

记录 30 次以上样本，计算 P50/P95：

1. `before_agent_start` 注入链路：P50 <= 450ms，P95 <= 1500ms。
2. `/api/v1/memories/search`：P50 <= 350ms，P95 <= 1200ms。
3. `/api/v1/memories`：P50 <= 220ms，P95 <= 700ms。

若超预算：

1. 先回退 `sharePolicy`/映射配置到最小集再测。
2. 仍超预算则回退 Task03 变更并保留日志样本。
