# MiniMem 开发指导：优先级重排与验证方案（2026-03-04）

## 0. 目标与硬约束

本文件作为后续开发唯一指导基线，目标：

1. 先完成配置统一，再推进功能增强。
2. 新增能力不允许导致主链路耗时大幅增长。
3. 每个改造项必须绑定冒烟验证与回测验证。

硬约束（必须满足）：

1. `/api/v1/memories/search`：P50 <= 350ms，P95 <= 1200ms。
2. OpenClaw `before_agent_start` 注入：P50 <= 450ms，P95 <= 1500ms。
3. `/api/v1/memories`（不含外部重模型抽取）：P50 <= 220ms，P95 <= 700ms。
4. 任一增强组件超时必须 `fail-open`，回退基础链路，不阻塞主流程。

## 1. 架构执行顺序（先架构后编码）

统一按三层推进：

1. 配置与策略层：`config.json` 单一配置源 + 明确优先级。
2. 集成适配层：OpenClaw/其他 Agent 的身份、group、权限、路由映射。
3. 记忆处理层：抽取、压缩、检索、rerank、skills 接入。

依赖关系：

1. `任务01` 是所有后续项前置依赖。
2. `任务02` 依赖 `任务01` 的配置优先级框架。
3. `任务05/任务07/任务06` 依赖统一配置开关和预算参数。

## 2. 逐条优先级重排（正式编号，含方案、冒烟、回测）

说明：

1. 优先级：P0 > P1 > P2。
2. 周期为单人净开发日估算，不含外部阻塞。

### 任务01（原 2.1）统一到 `config.json`（P0-1，最高优先级）

价值：高  
周期：8-12 天  
关键方案：

1. 新建配置结构：`server/models/retrieval/extractor/storage/security/integration`。
2. 启动策略：若 `config.json` 不存在，仅使用当前环境值生成初始化文件；存在则只读取 `config.json`。
3. 明确禁用旧策略：运行时不再读取 sqlite `app_config` 作为配置来源，写入也不再写 `app_config`。
4. `model-config` API 改为读写 `config.json`（必要项支持热加载）。

冒烟（每次提交）：

1. 启动服务，`GET /health`、`GET /api/v1/model-config` 正常。
2. `PUT /api/v1/model-config` 后重启，配置保持一致。
3. 已存在 sqlite `app_config` 但无 `config.json` 时，最终仍以新生成的 `config.json` 为准。

回测（阶段完成）：

1. 配置回归：对比初始化前后关键字段一致性（模型、检索、图谱开关）。
2. 功能回归：执行 `pytest -q tests/test_settings_paths.py tests/test_memory_route_policy.py tests/test_memory_route_payload.py`。

性能门禁：

1. 迁移后同数据集下检索 P95 回归不得超过 +10%。

性能门禁：

1. 不引入额外在线调用；模型决策仅配置解析，不得增加请求时延。

### 任务03（原 1.2）Agents/channels 映射与 group 路由（P0-3）

价值：高  
周期：5-7 天  
关键方案：

1. 增加 `senderMap/channelGroupMap/sharePolicy`。
2. `resolveScope` 先映射后回退默认 `groupStrategy`。
3. 写入 metadata：`agent_id/channel/task_id/trace_id`。
4. 插件按调用时读取最新 `pluginConfig`（不固化启动快照），支持 agent/channel 动态增删后即时生效。
5. 对新增未映射或已删除对象的陈旧映射，统一 `fail-open` 回退默认路由，不中断主流程。

冒烟：

1. 两个 agent 共享组可互相召回，私有组不可串读。
2. 缺映射时自动回退默认策略，不报错。
3. 新增 agent/channel 后，无需重装插件即可生效（无映射先回退，补映射后下一次请求命中新映射）。
4. 删除 agent/channel 后，请求继续可用，且不因历史映射导致报错。

回测：

1. 构造多 agent 多 group 测试集，验证 Recall@K 与串组率。
2. 新增并执行：`tests/test_openclaw_scope_mapping.py`。
3. 动态增删回放：增删后一轮请求内生效，且无需进程重启。

性能门禁：

1. scope 解析纯内存计算，请求附加耗时 P95 <= 30ms。

性能门禁：

## 9. 执行记录（2026-03-04，Task03 第二批修复：OpenClaw 空身份事件兜底）

问题现象（网关实测）：

1. `before_agent_start/agent_end` 事件在当前 OpenClaw 版本中，`agent_id/session_key/channel` 可能全部为空。
2. 导致 `resolveScope` 无法推断 agent，回退到 `openclaw-bot/default:openclaw-bot`，与 `--agent main` 预期不一致。

已落地修复：

1. 插件新增 `defaultAgentId` 配置（默认 `main`）：
   - 文件：`integrations/openclaw-plugin/index.ts`
   - 逻辑：`resolveScope` 解析顺序改为 `input.agent_id -> session_key -> defaultAgentId`。
2. 插件 schema 增加 `defaultAgentId`：
   - 文件：`integrations/openclaw-plugin/openclaw.plugin.json`
3. 安装脚本同步写入 `defaultAgentId=main`：
   - 文件：`integrations/openclaw-plugin/install.ps1`
   - 文件：`integrations/openclaw-plugin/install.sh`

回测结果：

1. `GWTEST-20260304-AGENTMAIN-1` 在 `user_id=main/group_id=default:main` 精确命中 `1` 条。
