&nbsp; 🔴 主要差距



&nbsp; 1. Semantic Boundary Detection（语义边界检测）

&nbsp;  - 论文: 使用 LLM 进行语义分割，检测话题转移，比固定启发式方法更好

&nbsp;  - 现状: MiniMem 使用基于时间/轮次的简单规则 (memory\_service.py:280-310)

&nbsp;  - 差距: 需要 LLM 驱动的语义分割器



&nbsp; 2. Narrative Synthesis（叙事合成）

&nbsp;  - 论文: 将对话重写为第三人称叙述，解决指代歧义

&nbsp;  - 现状: 直接存储原始对话，缺少 LLM 叙事合成步骤

&nbsp;  - 差距: 需要在 Phase I 添加叙事合成模块



&nbsp; 3. Scene-Level Profile Evolution（场景级画像演进）

&nbsp;  - 论文: 从 MemScene 摘要中提取画像更新，支持时间衰减和冲突追踪

&nbsp;  - 现状: 有基础画像更新，但缺少与 Scene 的紧密集成

&nbsp;  - 差距: 需要加强 Scene → Profile 的更新链路



&nbsp; 4. Foresight with Validity Intervals（前瞻记忆的时间有效性）

&nbsp;  - 论文: Foresight 带有 \[t\_start, t\_end] 有效期间隔，用于时序推理

&nbsp;  - 现状: 存储了 foresights 但缺少形式化的有效期区间处理

&nbsp;  - 差距: 需要实现时间有效性检查逻辑



&nbsp; 5. Agentic Verification \& Query Rewriting（智能体验证与查询重写）

&nbsp;  - 论文: 使用 LLM 验证器检查检索是否充分，不足时触发查询重写

&nbsp;  - 现状: 有两轮检索，但验证和重写逻辑简化

&nbsp;  - 差距: 需要更完善的 sufficiency checker



&nbsp; 6. Multi-hop \& Temporal Reasoning（多跳与时序推理）

&nbsp;  - 论文: 在 LongMemEval 上有显著提升 (知识更新 +20.6%)

&nbsp;  - 现状: 基础检索对齐，未针对复杂推理场景优化

&nbsp;  - 差距: 需要针对多跳/时序问题的专门优化



&nbsp; 7. Recency-Aware Profile Updates（基于最近性的画像更新）

&nbsp;  - 论文: 画像更新时考虑时间衰减，追踪冲突

&nbsp;  - 现状: 有冲突检测，但时间衰减机制较弱

&nbsp;  - 差距: 需要实现时间敏感的画像更新策略
